## Directory Purposes

**`src/openagentmesh/`:**
- Purpose: Core SDK implementation
- Contains: Python modules for the public API, internal mixins, models, error handling
- Key files: `_mesh.py` (entry point), `_invocation.py`, `_discovery.py`, `_models.py`, `_handler.py`

**`src/openagentmesh/cli/`:**
- Purpose: Command-line interface for mesh and agent operations
- Contains: Typer-based commands for mesh lifecycle, agent invocation, contract inspection
- Key files: `__init__.py` (Typer app), `mesh.py` (up/catalog), `agent.py` (call/stream/subscribe), `demo.py`

**`src/openagentmesh/demos/`:**
- Purpose: Reference implementations demonstrating SDK features
- Contains: Self-contained async scripts showing use patterns
- Key files: `hello_world.py`, `multi_agent.py`, `llm_tool_selection.py`, `load_balancing.py`

**`tests/`:**
- Purpose: Pytest test suite
- Contains: Unit, integration, and cookbook tests
- Organization: `test_*.py` for core features; `cli/` subdirectory for CLI tests; `cookbook/` subdirectory for doc sample validation

**`docs/`:**
- Purpose: Official user-facing documentation
- Contains: Markdown files processed by Zensical; code samples are the DX contract
- Organization: `concepts/` (agents, channels, discovery), `api/` (reference), `cookbook/` (recipes), `protocol/` (wire protocol)

**`km/`:**
- Purpose: Internal working material for design and architecture decisions
- Contains: ADRs, specs, notes, and workflow procedures
- Organization: `adr/` (decisions with status), `notes/` (brainstorm), `specs/` (detailed specs), `workflow/` (procedures)

## Naming Conventions

**Files:**
- Snake case: `my_module.py`, `test_mesh.py`
- Private modules (SDK internals) prefixed with underscore: `_mesh.py`, `_invocation.py`
- Avoid generic names like `utils.py` or `helpers.py`

**Directories:**
- Plural for collections: `docs/`, `tests/`, `demos/`
- Logical grouping: `src/openagentmesh/cli/`, `docs/concepts/`
- Workspaces in `.worktrees/` with descriptive names: `.worktrees/feature-async-contexts/`

**Classes:**
- PascalCase: `AgentMesh`, `AgentSpec`, `HandlerInfo`, `MeshError`
- Error subclasses end with "Error": `InvalidInput`, `HandlerError`, `ChunkSequenceError`
- Mixin classes end with "Mixin": `InvocationMixin`, `DiscoveryMixin`

**Functions:**
- Snake case: `inspect_handler()`, `compute_subject()`
- Private (internal) functions prefixed with underscore: `_connect()`, `_subscribe_agent()`
- Decorators are verbs: `@mesh.agent()`, `@app.callback()`

**Variables:**
- Snake case: `request_id`, `catalog_cache`, `is_streaming`
- Private attributes prefixed with underscore: `self._nc`, `self._agents`
- Constants in UPPER_CASE: `NATS_VERSION`, `_CATALOG_BUCKET`

**Types:**
- PascalCase for class names: `EchoInput`, `EchoOutput`
- Use `str | None` (PEP 604) instead of `Optional[str]`
- Type hints required on public API functions

## Where to Add New Code

**New Feature (e.g., new invocation method):**
- Primary code: `src/openagentmesh/_invocation.py` (or new mixin file)
- Tests: `tests/test_{feature_name}.py` (e.g., `tests/test_request_reply.py`)
- Documentation: `docs/concepts/invocation.md` or new doc file
- Reference demo: `src/openagentmesh/demos/{feature_name}.py`

**New Component/Module:**
- Implementation: `src/openagentmesh/_component.py` (private) or `src/openagentmesh/component/` (public subpackage)
- Export in: `src/openagentmesh/__init__.py` if public
- Tests: `tests/test_component.py`

**New CLI Command:**
- Implementation: `src/openagentmesh/cli/new_command.py`
- Registration: Add to app in `src/openagentmesh/cli/__init__.py`
- Tests: `tests/cli/test_new_command.py`

**Utilities/Helpers:**
- Shared helpers: `src/openagentmesh/_utils.py` (or split by domain: `_validation.py`, `_serialization.py`)
- Test fixtures: `tests/conftest.py` (pytest fixtures) or `tests/{module}_fixtures.py`

## Special Directories

**`.worktrees/`:**
- Purpose: Git worktrees for parallel feature branches
- Generated: Yes (user creates as needed)
- Committed: No (gitignored)
- Usage: `git worktree add .worktrees/feature-X feature/X-branch` for isolated dev

**`site/`:**
- Purpose: Built HTML documentation from Zensical
- Generated: Yes (run `uv run zensical build`)
- Committed: No (gitignored)
- Usage: Local preview with `uv run zensical serve`

**`km/adr/`:**
- Purpose: Decision records with status tracking
- Generated: No (hand-written)
- Committed: Yes (part of main branch)
- Index: `km/adr/index.md` lists all ADRs with status and branch assignments

**`km/workflow/`:**
- Purpose: Step-by-step procedures for development phases
- Generated: No (hand-written)
- Committed: Yes (part of main branch)
- Usage: Reference during session setup, implementation, and wrap-up
