# ADR-0056 admin UI — build waves

**Date:** 2026-07-18 (cron run 9). ADR-0056 amended the same day (ports, KV layout,
liveness, Watcher retirement, asset shipping — see the ADR's 2026-07-18 amendment).
Estimate from Stage-2 shaping: 4–6 sessions; each wave below is roughly one cron run.
The ADR stays at `spec`→`test`→`implemented` through waves 1–4 and reaches
`documented` only in wave 5, per the pipeline rules (docs land when the whole
feature is real).

## Wave 1 — mesh-side plumbing (run 9, this run)

- `websocket {}` listener in `render_mesh_server_conf` (mesh port + 1) and
  `EmbeddedNats` (free port; exposed as `ws_url`); `oam mesh up` prints the ws URL
  and records it for `oam ui`.
- `oam ui` command: stdlib static server for `_ui_assets/`, `GET /config.json`
  returning `{nats_ws_url}`, `--port/--host/--nats-ws-url` + `OAM_UI_HOST` /
  `OAM_NATS_WS_URL` env vars, free-port fallback, friendly error when assets are
  missing (source checkout), localhost-only default.
- Tests: config rendering, embedded ws port reachable (real websocket handshake),
  `oam ui` serves config.json + assets from a temp assets dir, port fallback.

## Wave 2 — frontend scaffold + registry (run 10) — DONE 2026-07-19

- `ui/` scaffold: Vite + React 18 + TS + Tailwind; pnpm via corepack
  (node 22 + corepack are present in the cloud sandbox; npm registry reachable —
  verified: `corepack pnpm@10 add` resolves and installs in <1s).
- **Mesh client = `@openagentmesh/sdk` via a workspace/file link to `sdk-ts/`**
  (verified 2026-07-18: the SDK picks `wsconnect` for ws:// URLs, keeps
  transport-node as a dynamic import browsers never load, and its `configUrl`
  connect option already implements the /config.json bootstrap). Do NOT
  reimplement call/stream/catalog-watch, and do NOT use the deprecated
  `nats.ws` package (upstream moved to @nats-io/nats-core + jetstream + kv,
  which the SDK uses).
- Screens: registry table + agent detail contract viewer (read-only).
- Vitest unit tests with a mocked SDK layer; CI job (ui: tsc + vitest + build).
- Shipped as planned (run 10): `ui/` scaffold, MeshProvider context (tests inject
  a fake client — no module mocking), useCatalog polls the SDK's KV-watch-warmed
  cache (2s), registry + detail screens with human/JSON toggle, 11 vitest tests,
  `ui` CI job. E2E verified in-sandbox with headless chromium against a real
  `oam mesh up` + `oam ui`: live catalog row, detail deep-link via SPA fallback,
  zero page errors. Events screen is a stub until wave 4.

## Wave 3 — invocation sandbox (runs 11–12) — DONE 2026-07-19

- @rjsf/core form from input schema; Call (request/reply with X-Mesh-Request-Id)
  and Stream (X-Mesh-Stream-Seq/End reassembly); error envelope rendering
  (taxonomy code + message, incl. not_available from gated agents).
- Shipped (run 11, verified + merged run 12): `InvokePanel` on the agent detail
  screen — rjsf form when an input schema exists, bare Call/Stream button when
  not; stream via the SDK's async iterator with a Stop button (AbortController);
  error box with taxonomy code badge, request_id, and per-code operator hints;
  source-only agents render no panel. Along the way: sdk-ts gained the
  `NotAvailable` mapping (no-responders + present in cached catalog → 
  not_available, ADR-0055 parity with Python). 8 new ui vitest tests (19 total),
  2 new sdk-ts tests (55 total).
- E2E verified in-sandbox (run 12, headless chromium against a real
  `oam mesh up` + `oam ui`): form generated from a Pydantic schema, call
  round-trip renders the reply, 4-chunk stream reassembles with status line,
  and a real kv_condition-gated agent produces the not_available error box
  (badge + gate hint + request_id). Zero page errors.

## Wave 4 — event feed + liveness (run 12) — DONE 2026-07-19

- Event feed screen (subject pattern subscribe, pause/clear).
- Registry status dot: mesh-instances KV watch + mesh.death.> subscription.
- Shipped (run 12): sdk-ts grew the two primitives the UI needed —
  `mesh.tap(pattern)` (wiretap: yields {subject, payload, isError}; decodes
  JSON or falls back to raw text; error envelopes yielded not thrown; ignores
  stream-end) and `mesh.instancesWatch()` (mesh-instances KV watch; replay
  coalesced into one initial snapshot; completes silently when the bucket is
  absent). UI: Events screen (pattern input default `mesh.>`, subscribe/
  unsubscribe via AbortController, pause buffers + resume flushes, clear,
  500-row cap, error-envelope highlighting) and registry status dots
  (useLiveness = instancesWatch ∪ mesh.death.> tap for instant offline).
  7 new sdk-ts tests (62 total), 11 new ui tests (30 total).
- Note: `>` as the pre-populated pattern (ADR text) was narrowed to `mesh.>`
  — a bare `>` tap receives the UI's own JS-API/inbox chatter and floods the
  feed with noise; record in the ADR amendment at wave-5 documentation time.
- E2E verified in-sandbox (run 12, chromium against a real `oam mesh up`,
  three hosts, `oam ui` production build): all dots live; SIGKILL of one
  host put `mesh.death.mortal` in the live feed within the 15s wait;
  pause/clear behaved; back on the registry the dead agent's row was
  REMOVED (health monitor deregisters — the gray dot covers only the
  pre-cleanup race and monitor-less meshes, worth stating in the docs);
  survivors stayed live; zero page errors.
