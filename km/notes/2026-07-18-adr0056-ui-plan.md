# ADR-0056 admin UI — build waves

**Date:** 2026-07-18 (cron run 9). ADR-0056 amended the same day (ports, KV layout,
liveness, Watcher retirement, asset shipping — see the ADR's 2026-07-18 amendment).
Estimate from Stage-2 shaping: 4–6 sessions; each wave below is roughly one cron run.
The ADR stays at `spec`→`test`→`implemented` through waves 1–4 and reaches
`documented` only in wave 5, per the pipeline rules (docs land when the whole
feature is real).

## Wave 1 — mesh-side plumbing (run 9, this run)

- `websocket {}` listener in `render_mesh_server_conf` (mesh port + 1) and
  `EmbeddedNats` (free port; exposed as `ws_url`); `oam mesh up` prints the ws URL
  and records it for `oam ui`.
- `oam ui` command: stdlib static server for `_ui_assets/`, `GET /config.json`
  returning `{nats_ws_url}`, `--port/--host/--nats-ws-url` + `OAM_UI_HOST` /
  `OAM_NATS_WS_URL` env vars, free-port fallback, friendly error when assets are
  missing (source checkout), localhost-only default.
- Tests: config rendering, embedded ws port reachable (real websocket handshake),
  `oam ui` serves config.json + assets from a temp assets dir, port fallback.

## Wave 2 — frontend scaffold + registry (run 10)

- `ui/` scaffold: Vite + React 18 + TS + Tailwind + nats.ws; pnpm via corepack
  (node 22 + corepack are present in the cloud sandbox; npm registry reachable).
- Mesh client module (connect via /config.json, KV watch on mesh-catalog/catalog,
  fetch mesh-registry/<name>).
- Screens: registry table + agent detail contract viewer (read-only).
- Vitest unit tests with a mocked nats.ws layer; CI job (ui: tsc + vitest + build).

## Wave 3 — invocation sandbox (run 11)

- @rjsf/core form from input schema; Call (request/reply with X-Mesh-Request-Id)
  and Stream (X-Mesh-Stream-Seq/End reassembly); error envelope rendering
  (taxonomy code + message, incl. not_available from gated agents).

## Wave 4 — event feed + liveness (run 12)

- Event feed screen (subject pattern subscribe, pause/clear).
- Registry status dot: mesh-instances KV watch + mesh.death.> subscription.

## Wave 5 — packaging, e2e, docs (run 13)

- Wheel packaging: build assets in the publish workflow into _ui_assets/
  (gitignored in the working tree); `[ui]` extra if any runtime dep appears
  (target: none).
- Playwright smoke e2e against a real `oam mesh up` + `oam ui` (chromium is
  preinstalled in the sandbox).
- Docs: cookbook/admin-ui.md (+ executable twin where testable), CLI reference,
  concepts touch-ups; ADR → documented; CHANGELOG.
